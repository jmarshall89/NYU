QueueBrowsers e ServerSessions

Utilize esta aba para reconfigurar as filas explorando o tempo de execução do cliente e para especificar novamente o comportamento das sessões em execução em um servidor de aplicativo.

Tamanho de recuperação do navegador da fila Especifica o número máximo de mensagens que o tempo de execução do cliente recuperará de uma vez, ao navegar o conteúdo do destino de uma fila. Padrão: 1.000

Tempo-limite de recuperação do navegador da fila (milissegundos) Especifica o tempo máximo que o tempo de execução do cliente esperará para recuperar as mensagens, ao navegar o conteúdo do destino de uma fila, antes de lançar uma exceção. Padrão: 60.000 milissegundo.

Carregar até o máximo de mensagens nas sessões do servidor Especifica que o ConnectionConsumer do Message Queue deve carregar até a quantidade de maxMessages mensagens na sessão do ServerSession. Não marque a caixa de seleção para especificar que o ConnectionConsumer do Message Queue deve carregar somente uma única mensagem de cada vez. Para obter mais informações, consulte a seção Recursos do servidor de aplicativo da especificação JMS.
