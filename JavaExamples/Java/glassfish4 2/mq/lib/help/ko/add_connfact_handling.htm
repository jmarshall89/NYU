연결 처리

연결 처리 탭을 사용하여 이 팩토리에서 만드는 연결을 구성합니다.

메시지 서버 주소 목록 (하나 이상의) 메시지 서버 주소가 쉼표로 구분된 목록을 지정합니다. 각 주소는 클라이언트 런타임이 연결할 수 있는 서로 다른 브로커 인스턴스를 가리킵니다. 목록의 각 주소는 호스트 이름, 호스트 포트 그리고 해당 연결에 대한 연결 서비스를 지정합니다.
메시지 서버 주소 구문에 대한 자세한 내용을 보려면 여기를 누르십시오

주소 목록 순서 메시지 서버 주소 목록상의 주소 순서(PRIORITY)에 따라 연결을 시도할 것인지, 임의의 순서(RANDOM)로 시도할 것인지를 지정합니다. 많은 클라이언트가 동일한 연결 팩토리를 사용하여 같은 연결을 시도할 경우 임의의 순서를 사용하면 클라이언트들이 같은 주소에 연결되는 것을 막을 수 있습니다.

주소 목록 반복 횟수 클라이언트 런타임이 연결을 설정(또는 재설정)하기 위해 메시지 서버 주소 목록을 반복하는 횟수를 지정합니다. 값이 -1이면 시도 횟수에 제한이 없습니다.

메시지 서버에 자동 다시 연결 활성화 값이 true이면 연결이 끊겼을 때 클라이언트 런타임이 메시지 서버(또는 메시지 서버 목록의 주소 목록)에 재연결을 시도합니다.

주소당 다시 연결 시도 수 클라이언트 런타임이 메시지 서버 주소 목록에서 다음 주소로 진행하기 전에 각 주소에 대해 재연결을 시도하는 횟수를 지정합니다. 값이 -1이면 시도 횟수에 제한이 없습니다.

주소당 다시 연결 간격(밀리초) 클라이언트 런타임이 메시지 서버 주소 목록에서 다음 주소로 진행하기 전에 각 주소에 대해 시도하는 재연결 간격(밀리초)을 지정합니다. 간격이 너무 짧으면 브로커가 복구할 시간 여유가 없어집니다. 너무 길면 재연결 시 너무 오래 지연될 수 있습니다.

연결 핑 간격(초) 클라이언트 런타임이 브로커에 대한 연결에 이상이 없는지 주기적으로 검사하는 간격을 초 단위로 지정합니다.
