Identificazione del client

Utilizzare questa scheda per specificare i valori predefiniti di nome utente e password e in che modo un client verrà identificato (ClientID) in un broker. I valori ClientId sono utilizzati principalmente per tenere traccia degli abbonati durable.

Nome utente predefinito Specifica il nome utente predefinito da utilizzare per l'autenticazione con il broker. Il valore predefinito è guest.

Password predefinita Specifica la password predefinita da utilizzare per l'autenticazione con il broker. Il valore predefinito è guest.

Disabilita API JMS setClientID() Specifica che l'applicazione client non può utilizzare le chiamate API per modificare ClientID. Generalmente, è consigliabile che l'amministratore controlli l'assegnazione dei ClientID.

Selezionando la casella, è possibile utilizzare il campo Configura ClientID come per specificare un ClientID per il client. Se tale valore non viene specificato, il valore predefinito che corrisponde all'indirizzo IP dell'host del client viene assegnato dal sistema. Questa operazione è utile agli sviluppatori, ma impedisce a un'applicazione client di creare più di una connessione: il sistema non consente due connessioni con lo stesso ClientID.

Configura ClientiID come Specifica un identificatore per il client. Se tale valore non viene specificato ed è stata disattivata la chiamata API che imposta il ClientID, il sistema assegna un valore predefinito che corrisponde all'indirizzo IP dell'host del client. Questa operazione è utile agli sviluppatori, ma impedisce a un'applicazione client di creare più di una connessione: il sistema non consente due connessioni con lo stesso ClientID.
